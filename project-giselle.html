GISelle — Safety-Aware Navigation & Pathfinding Platform

C++ · EZGL · A* · Multi-Destination Dijkstra · Simulated Annealing · Multi-threaded · U of T ECE297

GISelle (elle, French for “her”) is a safety-first GIS navigation platform developed for the University of Toronto ECE297: Software Design & Communication course. It extends shortest-path routing with safety-aware heuristics, emergency rerouting, and context-aware UI modes for women and vulnerable users. Awarded an A for algorithmic performance, implementation quality, and usability across all milestones.

Motivation

Standard routing optimizes for distance or time and does not model safety. GISelle adds:

  • Safety-aware cost functions — main-road bias, lighting, turn penalties.
  • Emergency rerouting — one-click redirect to nearest police station.
  • Shift-aware UI — day, night, and night-shift modes with high-contrast safety POIs (police, hospitals, transit).

Architecture & Milestones

Four phases: data pipeline and rendering, search and filtering, pathfinding, and delivery optimization.

Milestone 1: Data Integration & Rendering

Objective: GIS backend and renderer for interactive map display.

  • Input: Binary (`.bin`) map dumps.
  • Output: Structured graph (streets, intersections, POIs) with fast lookup for rendering and pathfinding.
  • Implementation: Custom parser → adjacency structures → EZGL 2D rendering with pan/zoom.

Milestone 2: Search & Context-Aware UI

Objective: Real-time street search and safety-oriented visual modes.

Data structures: Trie for prefix autocomplete; toggleable icon layers for police, hospitals, transit with day/night glyphs and tooltips.

UI modes: Default (full map); Dark (reduced glare); Night shift (emphasizes police, hospitals, transit; darker palette).

Milestone 3: A* Pathfinding

Objective: Time- and safety-aware routing between two intersections.

Cost model: Edge cost = travel time (not just length); main roads favored; turn penalties at intersections. Admissible heuristic for A* optimality.

Implementation: Priority queue over f(n) = g(n) + h(n); minimal node expansion.

Milestone 4: Traveling Courier Problem

Objective: Multi-pickup, multi-dropoff routes with depot and time limits via metaheuristics.

  • Precomputation: Multi-destination Dijkstra for travel-time matrix.
  • Optimization: Multi-start + 2-opt Simulated Annealing under 50 s cap.
  • Parallelism: std::thread for concurrent SA runs; keep best solution.

Safety Features

  • Find nearest police — one action reroutes to closest police station.
  • Regional helplines — toggle shows crisis/support numbers for current region.
  • Usability: SUS surveys and timed tasks (e.g. find safety button < 2 s).

Evaluation

  • Usability (SUS): 10 participants; high scores for confidence and ease of use.
  • Responsiveness: Majority located safety actions in < 2 s.
  • Courier (2-opt SA): ~10–15 s improvement in route time vs baseline.

Future Work

  • Street-light–aware edge costs for night routes.
  • Ride-hail API (e.g. Uber) with women/preferred-rider flow from map UI.
  • Crowdsourced safety: user-reported incidents or unsafe zones with moderation.
Safety shouldn’t be an afterthought — GISelle centers it in every route and UI decision.
